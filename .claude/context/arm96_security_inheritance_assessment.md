# arm96 二进制二次加固方案安全性继承评估（V0.45）

> 评估日期：2026-05-01  
> 评估对象：arm96 当前交付基线（V0.45，继承 V0.44 验收口径）  
> 信息来源：`.claude/README.md`、`.claude/instructions/project_entry.md`、`.claude/context/tango_hardening.md`

---

## 1. 评估范围与方法

本评估聚焦“安全性继承”本身，即：当前版本相对历史版本是否稳定继承了关键防护能力，以及这些能力在攻防中的实际价值与边界。

评估维度：
- 静态逆向抗性
- 单点补丁绕过抗性
- 运行期调试/注入对抗能力
- 跨版本继承稳定性
- 面向高投入定向攻击的抗性上限

---

## 2. 总体结论（Executive Summary）

当前 arm96（V0.45）在“无源码二进制二次加固”范式中属于高强度实现。  
相较传统“字符串抹除 + 分支补丁”方案，已形成较完整的纵深防御链。

**综合评级：A- ～ A（约 7.8～8.2 / 10）**。

结论含义：
- 对低到中等投入的静态分析、常规动态附加、单点 NOP patch，已有较强抑制力。
- 对高投入、长期、定向逆向团队，仍存在可被逐层拆解的现实风险，尚未达到稳态 A+。

---

## 3. 已继承且有效的核心安全能力

### 3.1 数据域绑定链已成型（高价值继承）

当前方案不再仅依赖控制流 gate-check，而是将多个因子融合进解密密钥，形成数据域失败：

- 平台寄存器特征融合（ISAR0）
- MAC OUI 融合
- EXPIRE_EPOCH 派生因子融合
- arm96server token 融合
- arm96server `.text` hash 融合（V0.40 起）

这显著提升了“单条分支 NOP 即绕过”的难度。

### 3.2 密码学基线提升（继承稳定）

从 AES-CBC 升级到 AES-128-GCM 后：
- 错误密钥或密文篡改会直接触发 tag 校验失败
- 失败行为更确定，减少“错误密钥偶然可执行”的灰区

### 3.3 运行期防护链持续有效

已形成“启动早期 + 常驻巡检 + 失效自愈”组合：
- loader 启动早期反调试
- arm96server PID tracking + TracerPid 轮询
- guardian 级联清理与重建

该链条对实际攻击路径（attach、替换、kill 守护）有连续覆盖，不是单点触发型防御。

### 3.4 混淆覆盖持续扩展（成本抬升明显）

V0.41 继续扩大 OLLVM 覆盖面，重点覆盖了解密与关键安全路径函数。  
这对“工具化、模板化、低经验逆向”非常有效，能显著拉长分析时间。

---

## 4. 主要残余风险与边界

### 4.1 “Done”更多是工程验收，不等于独立攻防证明

文档中多数完成项是功能与验收通过，说明工程闭环较完整；
但这不等于经过外部红队或高强度对抗测试后的抗性上限验证。

### 4.2 运行期防护受环境约束

反调试与守护机制大量依赖：
- `/proc` 可见性
- 内核行为
- SELinux/seccomp 策略
- 容器/设备运行时差异

环境漂移会影响检测与处置效果，存在平台相关性风险。

### 4.3 混淆不是最终安全边界

OLLVM、字符串加密、假数据填充本质是“提高成本”，不是不可逆防护。  
面对高投入定向逆向，最终仍可能被分层还原。

### 4.4 安全与可用性的长期张力

历史迭代显示：部分更激进策略曾因兼容性/稳定性问题被替代或降级。  
这说明当前强度是“安全-可用”平衡点，而非纯安全最优点。

- 静态逆向抗性：**8.0 / 10**
- 单点补丁绕过抗性：**8.5 / 10**
- 运行期调试对抗：**7.5 / 10**
- 跨版本继承稳定性：**8.0 / 10**
- 高投入定向攻击抗性：**6.5 ～ 7.0 / 10**

**综合：7.8 ～ 8.2 / 10（A- ～ A）**

## 6. 后续建议（按优先级）

### P0（优先）
1. 引入“对抗型验证”而非仅功能验收：
   - 固定样本做定期绕过演练
   - 记录平均绕过时间（TTR）与步骤复杂度
2. 建立“安全回归指标”并版本化：
   - 能否单点 patch 绕过
   - 是否可稳定 attach
   - 是否可替换关键组件不触发失败

### P1（次优先）
3. 将环境依赖项显式化为发布前检查清单：
   - SELinux、seccomp、kernel 行为差异
   - `/proc` 可见性与权限边界
4. 把“可用性例外”与“安全降级”绑定记录，防止隐性退化。

### P2（持续优化）
5. 对关键防线做“最小可替代设计”预案，减少单机制失效时的整体跌落。

---

## 7. 一句话结论

当前 arm96 二进制二次加固体系已经从“可被快速定位绕过”的 C 级范式，演进到“具备数据域绑定与运行期闭环”的 A-～A 级范式；其核心短板不在“有没有防护”，而在“面对高投入对手时的长期抗压证明”仍需补齐。
